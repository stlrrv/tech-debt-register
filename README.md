# Реестр задач

Расширенный реестр для управления техническим долгом и бизнес-задачами с системой приоритезации.

## Возможности

- **Два типа задач**: Технические и бизнес-задачи
- **Система приоритезации**: Автоматический расчет ценности и рекомендаций
- **Матрица ценности/усилий**: Визуализация задач в квадрантах
- **Экспорт/импорт**: JSON для публикации и совместного просмотра
- **Read-only режим**: Для разработчиков и заинтересованных сторон
- **Редактирование задач**: Возможность редактировать существующие задачи в режиме редактирования

## Структура данных

### Общие поля

- `type`: "business" или "tech"
- `title`: Название задачи
- `description`: Описание
- `direction`: web/1С/qa/common
- `size`: S/M/L/XL
- `processScore`: 1-3
- `delayScore`: 1-3
- `impactScore`: 1-3
- `deadlineScore`: 1-3 (Без срока = 1, Мягкий = 2, Жесткий = 3)
- `author`: Кто добавил
- `registryStatus`: new/candidate/planned/in_progress/done/deferred
- `createdAt`, `updatedAt`: Даты

### Для тех. задач

- `consequence`: Последствие
- `riskLevel`: critical/high/medium/low
- `burnHorizon`: 1_3m/3_6m/6_12m/someday
- `techCategory`: архитектура/производительность/тесты/зависимости/кодовая база/инфраструктура/безопасность/другое
- `needsDecomposition`: true/false
- `needsTechLead`: true/false

### Для бизнес-задач

- `businessGoal`: Цель
- `expectedOutcome`: Ожидаемый результат
- `initiator`: Руководитель/Продажи/Поддержка/Клиент/Внутренняя команда
- `externalDependencyNote`: Зависимости
- `definitionOfDone`: Критерий готовности

## Использование

1. **Заполнение реестра**: Добавляйте задачи через интерфейс
2. **Экспорт**: Нажмите "Экспорт в JSON" для создания `registry-data.json`
3. **Публикация**: Разместите `registry-data.json` в той же папке
4. **Просмотр**: Откройте `index.html` - приложение автоматически загрузит данные в read-only режиме
5. **Редактирование**: В read-only режиме нажмите "Войти в режим редактирования" для добавления/изменения задач

## Система приоритезации

Ценность = (Процесс × 3) + (Откладывание × 2) + Охват + Дедлайн

### Квадранты

- **Делаем первыми**: Высокая ценность + малые усилия
- **Планируем**: Высокая ценность + большие усилия
- **По остаточному**: Низкая ценность + малые усилия
- **Пока не делаем**: Низкая ценность + большие усилия
